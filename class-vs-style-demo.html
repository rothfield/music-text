üéµ Music Notation Examples

Method 1: Using Classes

<!-- CSS -->
.tuplet-3 { --tuplet-number: "3"; }
.tuplet-5 { --tuplet-number: "5"; }
.tuplet-7 { --tuplet-number: "7"; }

<!-- HTML -->
<span class="cm-music-note tuplet-3">1</span>
<span class="cm-music-note tuplet-5">1</span>
<span class="cm-music-note tuplet-7">1</span>

Method 2: Using Inline Styles

<!-- No predefined classes needed! -->
<span class="cm-music-note" style="--tuplet-number: '3'">1</span>
<span class="cm-music-note" style="--tuplet-number: '5'">1</span>
<span class="cm-music-note" style="--tuplet-number: '23'">1</span>

üß† When to Use Which?

‚úÖ Use class when:

  • You have fixed, reusable styles (like "red-text", "large-button")
  • You want maintainable CSS that can be changed globally
  • The styling is semantic (like "error", "warning", "primary")
  • You have a limited set of variations
<span class="error-message">Invalid input</span>
<span class="success-message">Saved successfully</span>

‚úÖ Use style when:

  • You have dynamic/computed values
  • Each element needs unique styling
  • You're setting CSS custom properties
  • You have infinite variations (like any tuplet number)
<span style="--tuplet-number: '11'; width: 200px;">Dynamic</span>
<span style="left: 45px; top: 123px;">Positioned</span>

üéØ CSS Selector Implications

Selecting by Class:

.red-text { color: red; } /* Selects class="red-text" */
.tuplet-3 { --tuplet-number: "3"; } /* Selects class="tuplet-3" */

Selecting by Style Attribute:

[style*="color: red"] { } /* Selects style="color: red" */
[style*="--tuplet-number"] { } /* Selects style="--tuplet-number: '3'" */

This is why the ultra-generalized selector works:

.cm-music-note[style*="--tuplet-number"]::before {
  content: var(--tuplet-number);
}

It finds elements that have BOTH the class AND the style attribute!

üèÜ Best of Both Worlds

You can combine class and style for maximum flexibility:

<span class="cm-music-note tuplet" style="--tuplet-number: '7'; --color: purple;">1</span>
  • class="cm-music-note" ‚Üí Basic note styling
  • class="tuplet" ‚Üí General tuplet behavior
  • style="--tuplet-number: '7'" ‚Üí Specific tuplet number
